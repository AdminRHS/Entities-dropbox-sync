# Шаблон для заповнення вакансій

## Опис

Ці шаблони використовуються для створення нових вакансій або оновлення існуючих у Strapi CMS.

## Файли

### 1. `vacancy-template.json`
Порожній шаблон з усіма полями структури. Використовується як основа для нової вакансії.

### 2. `vacancy-template-filled-example.json`
Шаблон з прикладами заповнення полів. Використовується як довідник для розуміння формату даних.

## Структура полів

### Основні поля

| Поле | Тип | Опис | Обов'язкове |
|------|-----|------|-------------|
| `id` | number | ID вакансії (автоматично при створенні) | ❌ |
| `title` | string | Назва вакансії | ✅ |
| `description` | string | Повний опис вакансії (Markdown) | ✅ |
| `subTitle` | string | Короткий підзаголовок | ⚠️ |
| `vacancySlug` | string | URL-слаг для вакансії | ✅ |
| `locale` | string | Мова (ru, en, ua, pl) | ✅ |
| `isHot` | boolean | Гаряча вакансія | ⚠️ |
| `cardDescription` | string | Опис для картки в списку | ✅ |
| `titleH1` | string | Заголовок H1 для сторінки | ⚠️ |
| `videoLink` | string | Посилання на відео (YouTube) | ⚠️ |
| `formTitle` | string | Заголовок форми заявки | ⚠️ |

### Секції з заголовками

| Поле | Опис |
|------|------|
| `productsTitle` | Заголовок секції продуктів/послуг |
| `responsibilityTitle` | Заголовок секції обов'язків |
| `toolsTitle` | Заголовок секції інструментів |

### Масиви даних

#### `products` - Продукти/Послуги
```json
{
  "id": null,
  "productTitle": "Назва продукту",
  "productText": "Опис продукту"
}
```

#### `responsibilities` - Обов'язки
```json
{
  "id": null,
  "responsibilityLi": "Опис обов'язку"
}
```

#### `tools` - Інструменти
```json
{
  "id": null,
  "toolText": "Назва інструменту"
}
```

### Категорії

```json
"categories": {
  "data": [
    {
      "id": 27,
      "attributes": {
        "categoryTitle": "Менеджеры",
        "categorySlug": "managers",
        "locale": "ru"
      }
    }
  ]
}
```

**Доступні категорії:**
- Менеджеры (managers) - ID: 27
- Разработчики (developers) - ID: 25
- Дизайн (design) - ID: (перевірити в Strapi)
- Інші (перевірити в Strapi)

## Як використовувати

### 1. Створення нової вакансії

1. Скопіюйте `vacancy-template.json`
2. Перейменуйте файл: `new-vacancy.json`
3. Заповніть всі обов'язкові поля
4. Перевірте формат JSON
5. Імпортуйте в Strapi

### 2. Оновлення існуючої вакансії

1. Відкрийте файл вакансії з папки `vacancy/`
2. Внесіть зміни
3. Перевірте формат JSON
4. Імпортуйте оновлення в Strapi

## Приклади заповнення

### Мінімальна вакансія

```json
{
  "attributes": {
    "title": "Frontend Developer",
    "description": "Опис вакансії",
    "vacancySlug": "frontend-developer",
    "locale": "ru",
    "cardDescription": "Короткий опис",
    "categories": {
      "data": [{"id": 25}]
    }
  }
}
```

### Повна вакансія з усіма секціями

Дивіться `vacancy-template-filled-example.json`

## Важливі примітки

1. **ID полів**: При створенні нової вакансії встановіть `id: null` для всіх вкладених об'єктів (products, responsibilities, tools)

2. **Slug**: `vacancySlug` повинен бути унікальним, використовуйте формат `kebab-case` (наприклад: `frontend-developer`)

3. **Locale**: Підтримувані мови: `ru`, `en`, `ua`, `pl`

4. **Markdown**: Поле `description` підтримує Markdown форматування

5. **Дата**: Поля `createdAt`, `updatedAt`, `publishedAt` заповнюються автоматично в Strapi

6. **Категорії**: Використовуйте існуючі ID категорій з Strapi

7. **Масиви**: Якщо секція не потрібна, залиште порожній масив `[]`

## Перевірка перед імпортом

- ✅ Всі обов'язкові поля заповнені
- ✅ JSON валідний (перевірте через JSON validator)
- ✅ Slug унікальний
- ✅ Категорії існують в Strapi
- ✅ Посилання на відео валідні
- ✅ Markdown коректно відформатований

## Підтримка

При виникненні питань звертайтесь до документації Strapi або команди розробки.
