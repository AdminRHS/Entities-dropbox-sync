## Ecosystem Relationships

### Responsibilities Ecosystem (Unified Architecture)

The Responsibilities ecosystem represents the core value proposition of LIBRARIES. As of 2025-11-17, it follows a unified architecture:

```
Responsibility = Action + Object + [Parameters]
```

**Example Relationships:**

1. **Create + Video Script + [Video Type, Duration, Style]**
   - Action: "Create" (Command Verb)
   - Object: "Video Script" (Video Generation Object)
   - Parameters: Style preferences, duration constraints

2. **Analyze + Lead Data + [Data Source, Metrics, Time Period]**
   - Action: "Analyze" (Process Verb)
   - Object: "Lead Data" (Lead Generation Object)
   - Parameters: Source filters, KPIs to measure

3. **Generate + AI Workflow + [Automation Type, Complexity]**
   - Action: "Generate" (Result Verb)
   - Object: "AI Workflow" (AI Automation Object)
   - Parameters: Workflow complexity, target platform

### Cross-Entity Dependencies

```
PROFESSIONS → PARAMETERS
  └─ Each profession has associated parameter sets

RESPONSIBILITIES → PROFESSIONS
  └─ Responsibilities assigned to specific professions

TOOLS → ACTIONS
  └─ Tools enable specific actions

OBJECTS → DEPARTMENTS
  └─ Objects owned/managed by departments

PARAMETERS → OBJECTS
  └─ 7,321 parameter-object mappings defined
```

## Navigation Tips

1. **Finding Responsibilities:** Start at `Responsibilities/Core/responsibilities_master.json`
2. **Finding Actions:** Use `Responsibilities/Actions/Actions_Master.json` (429 actions)
3. **Finding Objects:** Browse `Responsibilities/Objects/` by domain
4. **Finding Tools:** Check `Tools/` with category subfolders
5. **Finding Professions:** See `Professions/professions.json` master list
6. **Department Data:** Navigate to `DEPARTMENTS/By_Department/[DEPT]/`
