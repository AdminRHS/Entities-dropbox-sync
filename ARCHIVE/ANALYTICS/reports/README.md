## Research Report Schema

All reports follow the Research Report Schema (v1) with these sections:

1. **Report_Metadata** - Report ID, version, date, author, status
2. **Source_Prompts_and_Inputs** - Research prompts and inputs used
3. **Scope_and_Methodology** - What the report covers and how data was collected
4. **Findings** - Main body organized by topic, tool, workflow, or department
5. **Libraries_Updates** - Changes needed in LIBRARIES
6. **Task_Managers_Updates** - Changes needed in TASK_MANAGERS
7. **Gap_Analysis** - Discrepancies and missing entities
8. **Recommendations_and_Next_Actions** - Immediate, short-term, long-term actions
9. **Cross_References** - Related prompts, reports, libraries, templates

**See:** `ENTITIES/REPORTS/Research_Report_Schema.md` for complete schema definition

## Research Block Structure

Research blocks are 2-hour focused research sessions logged in `Research_Blocks/` folder.

**Block Template:**
- **Block ID:** `Block_###_[Topic].md`
- **Duration:** 2 hours
- **Topic:** Specific research topic (e.g., Attendance Dataset, Bonus Analytics)
- **Date:** ISO date
- **Status:** Planned / In Progress / Complete
- **Findings:** Key findings from the block
- **Next Steps:** Actions from the block

## Usage Guidelines

### Creating Research Reports

1. **Plan Research Block:** Create block log in `Research_Blocks/`
2. **Execute Research:** Follow Research Report Schema structure
3. **Document Findings:** Use schema sections to organize findings
4. **Store Report:** Save completed report in `Reports/` folder
5. **Update Index:** Add report to Finance research index

### Research Block Execution

1. **Set Block Duration:** 2 hours maximum
2. **Define Scope:** Clear research question or topic
3. **Document Process:** Log methodology and sources
4. **Capture Findings:** Document key findings and insights
5. **Identify Actions:** List immediate next steps
