# **Report: Testing GPT Agent for LinkedIn Old Connection Messaging Task**

## **1. Objective of the Task**

The goal of today’s task was to test the GPT-based LinkedIn agent designed to automate sending message templates to old connections as part of the **Old Connection Task**.

The agent was required to:

- Automatically locate old conversations (at least **6 months old or older**)  
- Insert the provided **message template** into the chat  
- Customize the template by:  
  - Adding the **lead’s first name**  
  - Adding the **manager’s name** (LinkedIn account sending the message)  
- Deliver these messages at a natural pace to avoid automation detection by LinkedIn  

Vlad from the AI department assisted with this experiment. We provided him access to a LinkedIn account specifically for testing.

---

## **2. Process Overview**

### **2.1. Task Preparation**

We outlined the following workflow for the GPT agent:

1. **Filter conversations** and identify leads whose communication history is **6+ months old**.  
2. **Sort and select** only the relevant message threads.  
3. **Insert the message template** into each selected chat.  
4. **Replace placeholders** by:
   - Adding the correct **first name of the lead**
   - Adding the correct **manager/account name** at the end  
5. **Send the message** and continue to the next conversation.  
6. **Include short delays** between messages to minimize automation detection.

### **2.2. Account Selection**

We chose an account with a **small number of connections** to minimize risk. Using such an account protects higher-value accounts from potential restrictions or blocks.

---

## **3. Testing Process and Observations**

### **3.1. Functionality**

During the test, we observed the GPT agent through its demo interface. Key observations:

The agent successfully:

- Found and recognized conversations older than six months  
- Inserted the designated message template correctly  
- Added only the lead’s **first name**, as instructed  
- Appended the correct **manager/account name**  
- Sent messages with cautious pacing to avoid detection  

In one session, the GPT agent sent **30 messages**, meaning it contacted 30 old leads.

### **3.2. Accuracy and Behavior**

The agent accurately followed all instructions:

- Proper sorting of conversations  
- Customized messaging for each lead  
- Maintained delays between sending messages  
- Consistent and stable behavior throughout the test  

Overall, the agent demonstrated high accuracy and operational reliability in this initial test.

---

## **4. Risks and Limitations Identified**

Despite the successful launch, several risks were identified:

### **4.1. LinkedIn Automation Restrictions**

LinkedIn has **strict anti-automation rules**, and accounts using automated tools may be restricted or blocked.  
This makes testing risky, especially for accounts with large networks.

### **4.2. IP Address Concerns**

Initially, we were concerned that the agent might use different IP addresses across sessions.

However, Vlad confirmed:

- The agent runs from a **real and consistent IP address**
- Future runs will use **the same IP** as the first test  

This reduces some risk but does not fully eliminate it.

### **4.3. Long-Term Uncertainty**

It remains unclear how LinkedIn will respond after:

- **2–3 repeated runs**
- Multiple days of automated messaging  
- Scaling the tool to larger accounts  

Although the first test completed successfully, future restrictions remain possible.

---

## **5. Next Steps**

### **Planned Actions for Monday**

- Run a **second test** using the same LinkedIn account and IP address  
- Monitor LinkedIn for any reaction to repeated automated activity  
- Evaluate:
  - Message delivery consistency  
  - Account safety  
  - Session stability  
  - Any warnings or restrictions  

If the account remains safe across multiple test cycles, the agent may be considered stable enough for broader use.

---

## **6. Conclusion**

The first test of the GPT LinkedIn agent for the Old Connection Task was **successful**, showing:

- ✔ Correct filtering of old messages  
- ✔ Proper template usage  
- ✔ Personalization (lead’s first name + manager’s name)  
- ✔ Natural pacing  
- ✔ 30 leads processed in one session  
- ✔ No immediate LinkedIn restrictions  

However, the **long-term safety** of using this tool remains uncertain.

Further testing is required to confirm:

- That LinkedIn will not block accounts after repeated use  
- That the agent remains stable across multiple sessions  
- That it is safe to use on accounts with a large number of connections  

For now, results look promising, but caution is still necessary moving forward.
