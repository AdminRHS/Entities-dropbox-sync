## Common Structure

All department prompts follow a similar structure:

1. **Purpose** - Department-specific report generation purpose
2. **Department Context** - Department overview, employees, projects, tools
3. **Base Context Integration** - Integration with MAIN PROMPT v4.md and PROMPT_Daily_Report_Collection.md
4. **Department-Specific Instructions** - Step-by-step processing instructions
5. **Report Template** - Department-specific report sections
6. **Department Vocabulary** - Department-specific terminology
7. **Project Matching** - Department-specific project identification
8. **Quality Standards** - Content and formatting requirements
9. **Example Usage** - Command examples for AI assistants
10. **Version History** - Prompt version tracking

## Integration Points

### Base Documents
- **MAIN PROMPT v4.md** - Located in each department folder (`/Nov25/{DEPARTMENT} Nov25/MAIN PROMPT v4.md`)
- **PROMPT_Daily_Report_Collection.md** - Main collection prompt orchestrator

### Data Sources
- **Prompt Logs** - Department-specific activity logs
- **Employee Directories** - From MAIN PROMPT v4.md
- **Project Directories** - From MAIN PROMPT v4.md
- **Daily Files** - Employee daily.md files in department folders

### Output Structure
- **Reports Folder:** `/Nov25/{DEPARTMENT} Nov25/Reports/{DAY}/`
- **Report File:** `Daily_Activity_Report_Nov{DAY}_2025.md`

---

## Usage Workflow

1. **Read Department Prompt Log** - Extract activities from department prompt log
2. **Process Daily Files** - Read employee daily.md files (if applicable)
3. **Process Call Transcripts** - Use MAIN PROMPT v4.md framework (if applicable)
4. **Generate Report** - Create comprehensive daily activity report
5. **Save Report** - Save to department Reports folder
