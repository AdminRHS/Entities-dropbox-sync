# Template 02: Executive Summary

**Purpose:** Provide high-level meeting overview in 3-5 bullet points
**Library Integration:** Minimal (focus on key outcomes)
**Priority:** Always include (every meeting)

---

## Template

```markdown
## Executive Summary

**Key Outcomes:** [1-2 sentence overview of main results]

**Main Discussion Points:**
- [Bullet 1: Primary topic discussed]
- [Bullet 2: Secondary topic discussed]
- [Bullet 3: Additional key topic]

**Critical Decisions:**
- [Decision 1 if any]
- [Decision 2 if any]

**Action Items Summary:** [X action items assigned to Y team members]

**Next Steps:** [What happens next - 1 sentence]
```

---

## Recognition Rules

### What to Include

**Key Outcomes:**
- Main accomplishment or conclusion
- Primary result or deliverable discussed
- Overall meeting success/progress

**Main Discussion Points:**
- 3-5 most important topics
- Prioritize decisions and action items
- Skip minor details

**Critical Decisions:**
- Only include if actual decisions made
- "Decided to..." not "Discussed whether to..."
- Skip if no decisions

**Action Items Summary:**
- Count of action items
- Number of people assigned
- General nature of tasks

**Next Steps:**
- Immediate follow-up
- Next meeting planned
- What happens before next sync

## Writing Guidelines

### Be Concise
- **Key Outcomes:** 1-2 sentences maximum
- **Main Discussion Points:** 3-5 bullets, one line each
- **Critical Decisions:** Only if decisions actually made
- **Action Items Summary:** One sentence with counts
- **Next Steps:** 1-2 sentences

### Focus on What Matters
**Include:**
- Decisions made
- Work approved/rejected
- Blockers identified
- Key insights
- Major discussions

**Exclude:**
- Detailed explanations (save for other sections)
- Minor topics
- Procedural discussion ("We started by...")
- Individual comments unless critical

### Use Active Language
- "Approved design" not "Design was approved"
- "Identified blocker" not "A blocker was identified"
- "Team decided" not "It was decided"

---

## Validation Checklist

- [ ] **Key Outcomes** accurately summarize main result (1-2 sentences)
- [ ] **Main Discussion Points** cover 3-5 most important topics
- [ ] **Critical Decisions** only include actual decisions (or omit section)
- [ ] **Action Items Summary** provides count and general nature
- [ ] **Next Steps** clearly state immediate follow-up
- [ ] **Length** is concise (executive can read in 30 seconds)
- [ ] **Accuracy** reflects actual meeting content
- [ ] **No details** saved for other sections (keep high-level)

---

## Common Mistakes

**Too Detailed:**
❌ "Olha presented 15 mockups showing different button styles, color variations, and hover states. The team discussed each option, considering accessibility, brand guidelines, and user feedback from previous designs..."

✅ "Reviewed button component variations; selected Option 3 for design system update"

**Too Vague:**
❌ "Team discussed various topics and made some decisions"

✅ "Approved Q4 marketing budget ($15K); decided to prioritize LinkedIn over email outreach"

**Missing Key Info:**
❌ "Good meeting, made progress"

✅ "Completed sprint planning (15 tasks); identified API blocker requiring vendor resolution before development starts"
