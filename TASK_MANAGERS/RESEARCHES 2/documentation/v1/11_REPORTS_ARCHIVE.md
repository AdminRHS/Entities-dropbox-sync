## Overview

The RESEARCHES 2 system generates multiple types of reports throughout the 7-phase workflow. Reports provide visibility into progress, identify gaps, validate integrations, and document completions.

### Key Statistics

- **5 Report Types** (Gap Analysis, Mapping, Phase Completion, Validation, Progress)
- **25+ Reports** generated per processed video
- **4 Report Locations** (Gap_Analysis, Library_Mapping, Phase_Reports, Validation)
- **70% Automated** report generation via Python scripts
- **30% Manual** reports requiring human analysis

### Quick Facts

- **Format:** Markdown (.md) files
- **Encoding:** UTF-8
- **Generated By:** Automated scripts + manual analysis
- **Frequency:** Per video + periodic summaries
- **Retention:** Permanent (archived, not deleted)
- **Purpose:** Progress tracking, validation, documentation

## Best Practices

### Report Generation Best Practices

**Do:**
‚úÖ Generate gap analysis immediately after Phase 2 (Extraction)
‚úÖ Generate mapping report immediately after Phase 4 (Integration)
‚úÖ Review automated reports before Phase 4 integration
‚úÖ Add manual context to automated reports (business value, notes)
‚úÖ Keep timestamped versions for analysis evolution tracking
‚úÖ Generate progress reports weekly/monthly for visibility

**Don't:**
‚ùå Skip gap analysis (critical for integration planning)
‚ùå Delete old reports (valuable historical data)
‚ùå Edit automated reports directly (re-run script instead)
‚ùå Generate reports without reviewing quality
‚ùå Forget to archive reports before major changes

---

### Report Review Best Practices

**Gap Analysis Review:**
1. Check Coverage Snapshot for high-priority gaps
2. Review Detailed Gaps section entity-by-entity
3. Validate next available IDs (provided by analyzer)
4. Prioritize by CRITICAL ‚Üí HIGH ‚Üí MED ‚Üí LOW
5. Note dependencies (Tools before Workflows)

**Mapping Report Review:**
1. Verify quality score ‚â•0.90 (target)
2. Check hierarchy validates (no orphaned entities)
3. Confirm all tools mapped to tasks
4. Validate Action‚ÜíObject chains complete
5. Review department distribution (reasonable allocation)

**Phase Completion Review:**
1. Verify all deliverables listed
2. Check metrics accurate (entity counts, time)
3. Confirm pending items documented
4. Review next phase planning (actionable)

---

## Troubleshooting

### Problem 1: Report Not Generated

**Symptom:**
```bash
python scripts/video_gap_analyzer.py Video_024
# No output file created
```

**Cause:** Script error or missing prerequisites

**Solution:**
```bash
# Check error output
python scripts/video_gap_analyzer.py Video_024 --debug

# Common issues:
# 1. Transcription file missing (Phase 1 incomplete)
# 2. LIBRARIES path incorrect (check config.py)
# 3. Permissions issue (check directory write access)

# Verify prerequisites:
ls 02_TRANSCRIPTIONS/Video_024.md
ls LIBRARIES/LBS_003_Tools/

# Run with verbose output
python scripts/video_gap_analyzer.py Video_024 --verbose
```

### Problem 4: Progress Report Empty

**Symptom:**
```bash
python scripts/generate_progress_report.py
# Output: "No data available"
```

**Cause:** VIDEO_PROGRESS_TRACKER.csv missing or empty

**Solution:**
```bash
# Check if tracker exists
ls VIDEO_PROGRESS_TRACKER.csv

# If missing, create from transcriptions
python scripts/update_video_progress.py --init

# Or manually create tracker CSV with headers:
echo "Video_Number,Title,Status,Current_Phase,Employee" > VIDEO_PROGRESS_TRACKER.csv

# Then run progress report again
python scripts/generate_progress_report.py
```
